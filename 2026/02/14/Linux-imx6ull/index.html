ubuntu文件系统结构

根目录是/。在嵌入式开发中知道这些文件都是干啥的还是挺重要的，不然有可能出现下好了驱动但是找不到的尴尬情况。。。

目录 作用
/bin 存放二进制可执行文件
/boot Ubuntu内核和启动文件
/dev 设备驱动文件
/etc 通常是一些配置文件（密码文件，profile设置环境变量）
/home 用户主文件夹
/lib & /lib64 库文件
/media 访问U盘
/proc 虚拟文件系统，以文件形式展示系统和进程的运行时信息。比如：
• /proc/cpuinfo → 查看CPU信息
• /proc/meminfo → 查看内存使用
• /proc/version → 查看内核版本
• /proc/uptime → 查看系统运行了多久
/tmp 临时文件
/usr unix software resource
/var 一些日志文件

绝对路径和相对路径

这个在Linux操作里也是经常要用，记住绝对路径是/根目录算起的路径；相对路径./是当前路径，../是上一层目录；../../../就是返回好几层。

ubuntu下的磁盘管理（重要）

需要准备一个空U盘
这里我发现一个问题，就是vmware识别不到我的U盘,关闭虚拟机打开虚拟机的那个文件夹，找到.vmx文件修改这一行，把FALSE改成TRUE。开机后一般就正常了。

可以看到/sdb和/sdb1是U盘的设备文件，/sdb表示U盘，/sdb1表示u盘的第一个分区

磁盘挂载与卸载

mouut umount
挂载是将储存设备（例如U盘）和一个现有文件夹关联起来（挂载点），之后在挂载点存东西就是在存储设备存东西。这样优点是非常灵活并且可以没有像windows那样26个字母限制，可以挂载无数个设备。比如上一节末尾输入ls sd* ; df后输出/dev/sdb1 122876032 768 122875264 1% /media/cyl/6B48-8C80这里挂载的就是U盘的第一个分区sdb1,ubuntu默认挂载的。

  • 卸载 umount /media/cyl/6B48-8C80 <- 这个是U盘的绝对路径，之前讲过U盘在/media里
  • 挂载 sudo mkdir udisk创建新文件夹待会挂载。mounut <设备> <挂载点>

udisk是门，通过门才能进入U盘。挂载后验证

挂载成功

磁盘分区

注意在磁盘分区前要卸载U盘，否则有可能导致数据损坏

命令 作用
m 显示帮助菜单
p 打印当前分区表（查看）
n 新建分区
d 删除分区
t 修改分区类型
w 写入并退出
q 不保存退出
  • 例如分区1GiB，1GiB = 1024 * 1024 *1024
    1GiB = 1024MiB
    1MiB = 1024KiB
    1KiB = 1024 字节这是计算机系统二进制换算(linux)
    硬盘是十进制:
    1 GB = 1000 MB
    1 MB = 1000 KB
    1 KB = 1000 字节
  • last sector = 2097152个扇区加上first sector(default 2048)

格式化

文件系统类型 常用命令 特点 适用场景
ext4 mkfs.ext4 /dev/sdb1 Linux默认格式，支持日志 Linux系统盘、数据盘
xfs mkfs.xfs /dev/sdb1 高性能，适合大文件 大型存储服务器
vfat/FAT32 mkfs.vfat /dev/sdb1 兼容性好，但单文件<4GB U盘、相机卡
ntfs mkfs.ntfs /dev/sdb1 Windows主流格式 需要和Windows交换数据的大U盘
exfat mkfs.exfat /dev/sdb1 兼容性好，无4GB限制 现代U盘、大文件传输

Ubuntu用户和用户组

用户

Linux是多用户操作系统，不同用户可以设置不同权限，ubuntu有三种用户

  • 初次用户
  • root用户
  • 普通用户
    用户记录在/etc/passwd文件内

用户组

为方便管理，将用户分组，可以设置非本组人员不能访问某些文件。每个用户可以属于多个不同的组。每个用户组有一个id，叫做GID。用户组信息存在/etc/group中。

创建用户和用户组

操作类别 命令 说明 示例
创建组 sudo groupadd 组名 创建新用户组 sudo groupadd developers
sudo groupadd -g GID 组名 指定GID创建组 sudo groupadd -g 1001 developers
删除组 sudo groupdel 组名 删除用户组 sudo groupdel developers
创建用户 sudo adduser 用户名 交互式创建用户（推荐） sudo adduser cyl
sudo useradd -m -s /bin/bash 用户名 非交互式创建用户 sudo useradd -m -s /bin/bash cyl
sudo useradd -u UID -g GID -m -s /bin/bash 用户名 指定UID/GID创建用户 sudo useradd -u 1002 -g 1001 -m -s /bin/bash cyl
密码管理 sudo passwd 用户名 设置或修改密码 sudo passwd cyl
删除用户 sudo deluser 用户名 删除用户（保留家目录） sudo deluser cyl
sudo deluser --remove-home 用户名 删除用户及家目录 sudo deluser --remove-home cyl
添加用户到组 sudo usermod -aG 组名 用户名 将用户添加到附加组 sudo usermod -aG developers cyl
sudo usermod -aG 组1,组2 用户名 添加到多个组 sudo usermod -aG developers,sudo cyl
从组移除 sudo gpasswd -d 用户名 组名 将用户从组中移除 sudo gpasswd -d cyl developers
授予sudo权限 sudo usermod -aG sudo 用户名 授予管理员权限 sudo usermod -aG sudo cyl
查看信息 groups 用户名 查看用户所属组 groups cyl
id 用户名 查看用户详细信息 id cyl
cat /etc/passwd 查看所有用户 cat /etc/passwd
cat /etc/group 查看所有组 cat /etc/group
getent passwd 用户名 查看特定用户 getent passwd cyl
getent group 组名 查看特定组 getent group developers

常用场景示例

场景 命令组合 说明
创建开发人员账号 bash<br>sudo groupadd developers<br>sudo adduser cyl<br>sudo usermod -aG developers cyl<br> 创建用户cyl并加入开发组
创建管理员账号 bash<br>sudo adduser cyl<br>sudo usermod -aG sudo cyl<br> 创建用户cyl并授予sudo权限
批量创建项目成员 bash<br>sudo groupadd project<br>for user in alice bob charlie; do<br> sudo adduser $user<br> sudo usermod -aG project $user<br>done<br> 批量创建用户并加入项目组
查看用户权限 bash<br>id cyl<br>groups cyl<br> 确认cyl的组和权限
重置用户环境 bash<br>sudo deluser --remove-home cyl<br>sudo adduser cyl<br> 完全重置cyl（删除后重建）

参数说明表

参数 适用命令 说明
-a usermod 追加（Append），与-G配合使用
-G usermod 指定附加组（Supplemental Groups）
-g useradd, groupadd 指定主组GID或创建组时指定GID
-m useradd 创建家目录（Make home directory）
-s useradd 指定登录Shell
-u useradd 指定用户UID
-d useradd, gpasswd 指定家目录（useradd）或从组删除（gpasswd）
--remove-home deluser 删除用户时同时删除家目录

重要注意事项

注意事项 说明
使用 -aG 而非 -G -aG 追加到组，-G 会覆盖现有组，可能导致用户从其他组丢失
重新登录生效 用户被添加到新组后，需要重新登录才能生效
删除组的前提 删除组前确保没有用户以此组为主组（主组不能删除）
主组 vs 附加组 每个用户必须有一个主组（通常是同名组），可以有多个附加组
密码复杂度 设置密码时建议使用强密码，包含大小写字母、数字和特殊字符
权限最小化 只授予用户必要的最小权限，避免过度授权

Ubuntu文件权限

对于一个文件有三种状态：读r,写w,执行x

权限 二进制 数字 说明
--- 000 0 无权限
--x 001 1 仅执行
-w- 010 2 仅写入
-wx 011 3 写入+执行
r-- 100 4 仅读取
r-x 101 5 读取+执行
rw- 110 6 读取+写入
rwx 111 7 读取+写入+执行
数字权限 字符权限 适用场景
777 rwxrwxrwx 完全开放（不安全）
755 rwxr-xr-x 目录默认权限，所有者全权，其他人只读执行
750 rwxr-x--- 共享目录，同组可访问
700 rwx------ 私有目录
644 rw-r--r-- 文件默认权限
640 rw-r----- 共享文件，同组可读
600 rw------- 私有文件

例如：

drwxr xr x

ubuntu/linux文件权限修改

修改文件权限命令–chmod命令

数字方式

命令 权限 说明
chmod 755 文件 rwxr-xr-x 目录默认权限
chmod 644 文件 rw-r--r-- 文件默认权限
chmod 700 文件 rwx------ 私有目录
chmod 600 文件 rw------- 私有文件
chmod 750 文件 rwxr-x--- 共享目录
chmod 640 文件 rw-r----- 共享文件
chmod -R 755 目录 rwxr-xr-x 递归修改

符号方式

命令 含义
chmod u+x 文件 所有者加执行
chmod g-w 文件 所属组删写
chmod o+r 文件 其他用户加读
chmod a+x 文件 所有人加执行
chmod u=rwx,g=rx,o=r 文件 精确设置(754)

修改所有者和组命令 – chown/chgrp

chown 命令

命令 含义
sudo chown cyl 文件 修改所有者为cyl
sudo chown :cyl 文件 修改所属组为cyl
sudo chown cyl:cyl 文件 同时修改所有者和组为cyl
sudo chown cyl: 文件 同上（简写）
sudo chown -R cyl:cyl 目录 递归修改目录及内容

chgrp 命令

命令 含义
sudo chgrp cyl 文件 仅修改所属组为cyl
sudo chgrp -R cyl 目录 递归修改目录的所属组

常用组合

命令 含义
sudo chown -R cyl:cyl /home/cyl 修复用户家目录权限
sudo chown --reference=file1 file2 参考file1的所有者设置file2
